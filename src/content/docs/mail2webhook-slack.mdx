When `format` is set to `slack`, incoming emails are converted into Slack messages using the [Incoming Webhooks](https://api.slack.com/messaging/webhooks) API. No custom payload handling required on your end.

---

## Get the Webhook URL

<Steps>

1. Go to [api.slack.com/apps](https://api.slack.com/apps) and click **Create New App** → **From scratch**. Give it a name and select your workspace.

2. In the app settings sidebar, click **Incoming Webhooks** and toggle it **On**.

3. Click **Add New Webhook to Workspace**, select the channel where messages should appear, then click **Allow**.

4. Copy the generated URL — it looks like:
   ```
   https://hooks.slack.com/services/T00000000/B00000000/YOUR_WEBHOOK_PATH
   ```

</Steps>

---

## Configure the Webhook

```sh
curl 'https://api.echovalue.dev/webhook' \
  -H 'x-token: mytoken' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://hooks.slack.com/services/T00000000/B00000000/YOUR_WEBHOOK_PATH",
    "format": "slack"
  }'
```

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `url` | string | Yes | Your Slack Incoming Webhook URL |
| `format` | string | Yes | Must be `"slack"` |
| `options` | object | No | Not used for Slack |

---

## Message Preview

Each email arrives in Slack as:

```
*Subject line*
From: Sender Name <sender@example.com>
Email body text
```

:::note
The plain text body is used. HTML is stripped automatically.
:::
