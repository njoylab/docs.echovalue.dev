`PUT https://api.echovalue.dev/webhook/<webhookId>`

Replaces the full webhook document for the specified `webhookId`. `url` is required.

`webhookId` is a path parameter and is not allowed in the request body.

:::note
**Cost:** 1 credit for each `PUT` or `PATCH` call.
:::

## Replace Request

**Headers:**

| Header | Description |
| ------ | ----------- |
| `x-token` | Your API token |
| `Content-Type` | Must be `application/json` |

**Body** (`application/json`):

| Field | Type | Description | Required |
| ----- | ---- | ----------- | -------- |
| `url` | string | Callback URL; must be HTTPS and publicly reachable | Yes |
| `headers` | object | Custom headers to include in the callback request | No |
| `format` | string | Native message format | No |
| `template` | object | Custom template when `format` is `custom` | No |
| `options` | object | Format-specific options | No |
| `includeAttachments` | boolean | Whether attachments are included in payloads. Relevant for webmail-triggered webhooks. | No |
| `schedule` | object | Full scheduling configuration for cron-triggered webhooks | No |

**`schedule` fields:**

| Field | Type | Description | Required |
| ----- | ---- | ----------- | -------- |
| `cron` | string | 5-field cron expression: `minute hour day month weekday` | Yes |
| `timezone` | string | IANA timezone name | Yes |
| `active` | boolean | Whether scheduled delivery is enabled. Default: `true`. | No |

:::caution
Cron schedules must not create gaps longer than 30 days between runs. The same 30-day limit applies from update time to the next run.
:::

## Replace Response

Response body and response headers are the same as [Create Webhook](/mail2webhook/create/).

## Replace Status Codes

| Status | Meaning |
| ------ | ------- |
| `200` | Webhook replaced |
| `400` | Invalid URL or malformed request |
| `401` | Invalid token |
| `402` | Insufficient credits |
| `404` | Webhook not found |
| `500` | Internal server error |

```go
package main

import (
    "bytes"
    "encoding/json"
    "net/http"
)

func main() {
    payload := map[string]interface{}{

        "schedule": map[string]interface{}{
            "cron": "0 9 * * *",
            "timezone": "Europe/Rome",

    }
    jsonData, _ := json.Marshal(payload)

    req.Header.Set("Content-Type", "application/json")

    client := &http.Client{}
    client.Do(req)
}
```

## Patch Endpoint

`PATCH https://api.echovalue.dev/webhook/<webhookId>`

Updates only the fields you provide.

`webhookId` is a path parameter and is not allowed in the request body.

**Patch semantics:**

| Field | Behavior |
| ----- | -------- |
| `url`, `format`, `includeAttachments` | Replaced when provided |
| `headers` | Replaces the full headers object. Use `{}` to clear it. |
| `options` | Replaces the full options object. Use `{}` to clear it. |
| `template` | Replaces the template. Use `null` to remove it. |
| `schedule` | Updates only the provided schedule fields. Use `null` to remove scheduling entirely. |

## Patch Request

**Headers:**

| Header | Description |
| ------ | ----------- |
| `x-token` | Your API token |
| `Content-Type` | Must be `application/json` |

## Patch Response

Response body and response headers are the same as [Create Webhook](/mail2webhook/create/).

## Patch Status Codes

| Status | Meaning |
| ------ | ------- |
| `200` | Webhook updated |
| `400` | Invalid URL or malformed request |
| `401` | Invalid token |
| `402` | Insufficient credits |
| `404` | Webhook not found |
| `500` | Internal server error |
