`GET https://api.echovalue.dev/webhook`

Returns all configured webhooks for your wallet.

:::note
**Cost:** 1 credit
:::

## Request

**Headers:**

| Header | Description |
| ------ | ----------- |
| `x-token` | Your API token |

## Response

`200 OK` â€” JSON object.

```json
{
  "webhooks": [
    {
      "id": "slack",
      "webhook": "https://hooks.slack.com/services/T00/B00/XXX",
      "headers": {},
      "format": "slack",
      "options": {},
      "includeAttachments": true,
      "email": "r_abcd1234@hook.echovalue.dev",
      "hash": "a1b2c3d4e5f6..."
    },
    {
      "id": "0",
      "webhook": "https://yourdomain.com/webhook",
      "headers": { "Authorization": "<redacted>" },
      "options": {},
      "includeAttachments": true,
      "email": "r_wxyz6789@hook.echovalue.dev",
      "hash": "a1b2c3d4e5f6..."
    }
  ]
}
```

| Field | Type | Description |
| ----- | ---- | ----------- |
| `webhooks` | array | Array of webhook configurations |

**Webhook entry fields:**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `id` | string | Webhook identifier (e.g. `slack`, `0`) |
| `webhook` | string | Your configured callback URL |
| `headers` | object | Custom headers (values are redacted) |
| `format` | string | Configured native format, if any (e.g. `slack`, `telegram`, `custom`) |
| `template` | object | Custom template object (only present when format is `custom`) |
| `options` | object | Format-specific options (sensitive values like `routing_key` are redacted as `***`) |
| `includeAttachments` | boolean | Whether attachments are included in webhook payload |
| `email` | string | Opaque mailbox address that triggers this webhook (`<mailboxId>@hook.echovalue.dev`) |
| `hash` | string | SHA256 hash of your token â€” use this to identify incoming payloads |

| Status | Meaning |
| ------ | ------- |
| `200` | Configuration returned |
| `401` | Invalid token |
| `402` | Insufficient credits |
