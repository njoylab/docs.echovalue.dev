Use `GET /webhook` to list all configured webhooks for your wallet and `GET /webhook/<webhookId>` to retrieve a specific one.

## List Webhooks

`GET https://api.echovalue.dev/webhook`

Returns all configured webhooks for your wallet.

:::note
**Cost:** 1 credit
:::

## Request

**Headers:**

| Header | Description |
| ------ | ----------- |
| `x-token` | Your API token |

<Tabs syncKey="lang">
<TabItem label="cURL">

```sh
curl 'https://api.echovalue.dev/webhook' \
  -H 'x-token: mytoken'
```

</TabItem>
<TabItem label="JavaScript">

```js
fetch('https://api.echovalue.dev/webhook', {
  headers: { 'x-token': 'mytoken' }
})
.then(response => response.json())
.then(data => console.log(data));
```

</TabItem>
<TabItem label="Python">

response = requests.get('https://api.echovalue.dev/webhook',

## List Response

`200 OK` — JSON object.

```json
{
  "webhooks": [
    {
      "webhookId": "slack",
      "name": "slack",
      "webhook": "https://hooks.slack.com/services/T00/B00/XXX",
      "headers": {},
      "format": "slack",
      "options": {},
      "includeAttachments": true,
      "email": "r_abcd1234@hook.echovalue.dev",
      "hash": "a1b2c3d4e5f6..."
    },
    {
      "webhookId": "default",
      "name": "primary",
      "webhook": "https://yourdomain.com/webhook",
      "headers": { "Authorization": "<redacted>" },
      "options": {},
      "includeAttachments": true,
      "email": "r_wxyz6789@hook.echovalue.dev",
      "hash": "a1b2c3d4e5f6..."
    }
  ]
}
```

| Field | Type | Description |
| ----- | ---- | ----------- |
| `webhooks` | array | Array of webhook configurations |

**Webhook entry fields:**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `webhookId` | string | Public webhook identifier used for get/update/delete/test operations |
| `name` | string | Webhook name (optional label) |
| `webhook` | string | Your configured callback URL |
| `headers` | object | Custom headers |
| `format` | string | Configured native format, if any (e.g. `slack`, `telegram`, `custom`) |
| `template` | object | Custom template object (only present when format is `custom`) |
| `options` | object | Format-specific options (sensitive values like `routing_key` are redacted as `***`) |
| `includeAttachments` | boolean | Whether attachments are included in webhook payload |
| `email` | string | Opaque mailbox address that triggers this webhook |
| `hash` | string | SHA256 hash of your token — use this to identify incoming payloads |

**Response headers:**

| Header | Description |
| ------ | ----------- |
| `x-balance` | Wallet balance after this call |
| `x-cost` | Tokens consumed (1) |

See [Response Headers](/response-headers) for details.

| Status | Meaning |
| ------ | ------- |
| `200` | Configuration returned |
| `401` | Invalid token |
| `402` | Insufficient credits |

## Single Webhook Response

`200 OK` — JSON object.

```json
{
  "webhookId": "slack",
  "name": "slack",
  "webhook": "https://hooks.slack.com/services/T00/B00/XXX",
  "headers": {},
  "format": "slack",
  "options": {},
  "includeAttachments": true,
  "email": "r_abcd1234@hook.echovalue.dev",
  "hash": "a1b2c3d4e5f6..."
}
```

| Field | Type | Description |
| ----- | ---- | ----------- |
| `webhookId` | string | Public webhook identifier used for update/delete/test operations |
| `name` | string | Webhook name (optional label) |
| `webhook` | string | Your configured callback URL |
| `headers` | object | Custom headers (values are redacted) |
| `format` | string | Configured native format, if any (e.g. `slack`, `telegram`, `custom`) |
| `template` | object | Custom template object (only present when format is `custom`) |
| `options` | object | Format-specific options (sensitive values like `routing_key` are redacted as `***`) |
| `includeAttachments` | boolean | Whether attachments are included in webhook payload |
| `email` | string | Opaque mailbox address that triggers this webhook |
| `hash` | string | SHA256 hash of your token — use this to identify incoming payloads |

**Response headers:**

| Header | Description |
| ------ | ----------- |
| `x-balance` | Wallet balance after this call |
| `x-cost` | Tokens consumed (1) |

See [Response Headers](/response-headers) for details.

| Status | Meaning |
| ------ | ------- |
| `200` | Webhook returned |
| `401` | Invalid token |
| `402` | Insufficient credits |
| `404` | Webhook not found |
