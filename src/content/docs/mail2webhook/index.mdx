Mail2Webhook supports two trigger modes:

- `webmail`: echoValue assigns an opaque mailbox address in the form `<mailboxId>@hook.echovalue.dev`. Emails sent to that address are forwarded as JSON `POST` requests to your configured webhook URL.
- `cron`: echoValue calls your webhook on a recurring schedule defined by a 5-field cron expression plus an IANA timezone.

Each webhook is managed through a public `webhookId`. If you omit it when creating a webhook, echoValue uses `default`.

## Trigger Modes

### Webmail-triggered webhooks

- Return an opaque `email` address in API responses.
- Support `includeAttachments`.
- Deliver the email-style payload documented in [Webhook Payload](/mail2webhook/payload/).

### Cron-triggered webhooks

- Are enabled by sending a `schedule` object when creating or updating a webhook.
- Do not return an `email` mailbox address.
- Deliver a scheduled-run payload instead of the inbound email payload.
- Cost 1 credit for each scheduled execution.

:::caution
Cron schedules must not produce gaps longer than 30 days between runs. The same 30-day limit also applies from creation time to the first scheduled run.
:::

## Native Format Support

Set `format` when configuring your webhook to have echoValue transform incoming emails into the native payload format for your platform â€” no custom handling required on your end.

| Format | Platform |
| ------ | -------- |
| `slack` | [Slack Incoming Webhooks](/mail2webhook/formats/slack/) |
| `discord` | [Discord Webhooks](/mail2webhook/formats/discord/) |
| `teams` | [Microsoft Teams](/mail2webhook/formats/teams/) |
| `telegram` | [Telegram Bot API](/mail2webhook/formats/telegram/) |
| `pagerduty` | [PagerDuty Events API v2](/mail2webhook/formats/pagerduty/) |
| `custom` | [Custom JSON template](/mail2webhook/formats/custom/) |

Without a `format`, the raw email payload is forwarded as-is (see [Webhook Payload](/mail2webhook/payload)).

## Webhook IDs

- `webhookId` is public and stable for management operations.
- The email address remains opaque and is returned by the API response.
- The returned `email` mailbox may not resemble the chosen `webhookId`.
