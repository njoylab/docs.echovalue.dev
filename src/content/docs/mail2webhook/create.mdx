`POST https://api.echovalue.dev/webhook`

Creates a new webhook. A webhook can be triggered by inbound email (`webmail`) or by a recurring cron schedule.

:::note
**Cost:** 1 credit.
:::

## Request

**Headers:**

| Header | Description |
| ------ | ----------- |
| `x-token` | Your API token |
| `Content-Type` | Must be `application/json` |

**Body** (`application/json`):

| Field | Type | Description | Required |
| ----- | ---- | ----------- | -------- |
| `webhookId` | string | Public webhook identifier used for management endpoints. Defaults to `default` when omitted. | No |
| `url` | string | Callback URL; must be HTTPS and publicly reachable | Yes |
| `headers` | object | Custom headers to include in the callback request | No |
| `format` | string | Native message format: `slack`, `discord`, `teams`, `telegram`, `pagerduty`, `custom` | No |
| `template` | object | JSON template for `custom` format. Max 2048 bytes, 5 nesting levels. | No |
| `options` | object | Format-specific options such as `chat_id` or `routing_key` | No |
| `includeAttachments` | boolean | Include email attachments in payloads. Default: `true`. | No |
| `schedule` | object | Recurring schedule for cron-triggered webhooks. See fields below. | No |

**`schedule` fields:**

| Field | Type | Description | Required |
| ----- | ---- | ----------- | -------- |
| `cron` | string | 5-field cron expression: `minute hour day month weekday` | Yes |
| `timezone` | string | IANA timezone used to evaluate the cron expression, for example `Europe/Rome` | Yes |
| `active` | boolean | Whether scheduled delivery is enabled. Default: `true`. | No |

Mailbox-style webhooks return an opaque `email` address that may not match the chosen `webhookId`. Scheduled webhooks do not return an `email` field.

:::caution
For scheduled webhooks, the cron expression must not create gaps longer than 30 days between runs. The first run must also be within 30 days from creation.
:::

:::note
Each scheduled execution costs 1 credit. Email-triggered deliveries still cost 2 or 5 credits depending on attachments.
:::

For format-specific setup see [Slack](/mail2webhook/formats/slack/), [Discord](/mail2webhook/formats/discord/), [Teams](/mail2webhook/formats/teams/), [Telegram](/mail2webhook/formats/telegram/), [PagerDuty](/mail2webhook/formats/pagerduty/), and [Custom](/mail2webhook/formats/custom/).

## Response

`200 OK` — JSON object.

```json
{
  "webhookId": "daily-sync",
  "webhook": "https://example.com/hooks/daily-sync",
  "headers": {},
  "format": "",
  "options": {},
  "schedule": {
    "cron": "0 9 * * *",
    "timezone": "Europe/Rome",
    "active": true,
    "nextRunAt": "2026-04-24T07:00:00.000Z"
  },
  "hash": "a1b2c3d4e5f6..."
}
```

| Field | Type | Description |
| ----- | ---- | ----------- |
| `webhookId` | string | Public webhook identifier for management operations |
| `webhook` | string | Configured callback URL |
| `headers` | object | Custom headers |
| `format` | string | Native format, if configured |
| `template` | object | Custom template object when `format` is `custom` |
| `options` | object | Format options; sensitive values are redacted |
| `includeAttachments` | boolean | Whether attachments are included in webhook payloads. Returned only for webmail-triggered webhooks. |
| `schedule` | object | Schedule metadata for cron-triggered webhooks, including `cron`, `timezone`, `active`, and runtime fields such as `nextRunAt` |
| `email` | string | Opaque mailbox address that receives email for this webhook. Returned only for webmail-triggered webhooks. |
| `hash` | string | SHA256 hash of your token |

**Response headers:**

| Header | Description |
| ------ | ----------- |
| `x-balance` | Wallet balance after this call |
| `x-cost` | Credits consumed |

See [Response Headers](/response-headers/) for details.

## Status Codes

| Status | Meaning |
| ------ | ------- |
| `200` | Webhook created |
| `400` | Invalid URL or malformed request |
| `401` | Invalid token |
| `402` | Insufficient credits |
| `409` | `webhookId` already exists |
| `500` | Internal server error |

<?php
$data = json_encode([

<TabItem label="Go">

```go
package main

import (
    "bytes"
    "encoding/json"
    "net/http"
)

func main() {
    payload := map[string]interface{}{
        "webhookId": "daily-sync",
        "url": "https://example.com/hooks/daily-sync",
        "schedule": map[string]interface{}{
            "cron": "0 9 * * *",
            "timezone": "Europe/Rome",
        },
    }
    jsonData, _ := json.Marshal(payload)

    req, _ := http.NewRequest("POST", "https://api.echovalue.dev/webhook", bytes.NewBuffer(jsonData))
    req.Header.Set("x-token", "mytoken")
    req.Header.Set("Content-Type", "application/json")

    client := &http.Client{}
    client.Do(req)
}
```
