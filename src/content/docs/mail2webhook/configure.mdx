`POST https://api.echovalue.dev/webhook`

Creates a new webhook when `mailboxId` is omitted, or updates an existing mailbox-owned webhook when `mailboxId` is provided.

:::note
**Cost:** 1 credit
:::

## Request

**Headers:**

| Header | Description |
| ------ | ----------- |
| `x-token` | Your API token |
| `Content-Type` | Must be `application/json` |

**Body** (`application/json`):

| Field | Type | Description | Required |
| ----- | ---- | ----------- | -------- |
| `mailboxId` | string | Opaque mailbox identifier. Omit on create, include on update. | No |
| `name` | string | Human-friendly webhook name. Optional on create/update. | No |
| `url` | string | Callback URL (must be HTTPS, publicly reachable) | Yes |
| `headers` | object | Custom headers to include in the callback request | No |
| `format` | string | Native message format: `slack`, `discord`, `teams`, `telegram`, `pagerduty`, `custom` | No |
| `template` | object | JSON template for `custom` format. Supports placeholders like `{{subject}}`, `{{from.name}}`, `{{attachments[0].filename}}`, fallback chains `{{text\|html}}`, and literal fallbacks `{{sender\|"unknown"}}`. Max 2048 bytes, 5 nesting levels. | No |
| `options` | object | Format-specific options (e.g. `chat_id` for Telegram, `routing_key` for PagerDuty). Max 10 keys, values up to 500 chars. | No |
| `includeAttachments` | boolean | Include email attachments in webhook payload (default: `true`). If `false`, attachments are stripped from the payload. When email contains attachments, costs 5 tokens; without attachments costs 2 tokens. | No |

Use `GET /webhook` to retrieve each webhook's `mailboxId` for update, delete, and test operations.

For platform-specific setup guides see: [Slack](/mail2webhook-slack), [Discord](/mail2webhook-discord), [Teams](/mail2webhook-teams), [Telegram](/mail2webhook-telegram), [PagerDuty](/mail2webhook-pagerduty).

package main

import (
    "bytes"
    "encoding/json"
    "net/http"
)

func main() {
    payload := map[string]interface{}{
        "name":   "slack",
        "url":    "https://hooks.slack.com/services/T00/B00/XXX",

    }
    jsonData, _ := json.Marshal(payload)

    client := &http.Client{}
    client.Do(req)
}

## Response

`200 OK` — JSON object.

```json
{
  "name": "slack",
  "webhook": "https://hooks.slack.com/services/T00/B00/XXX",
  "headers": {},
  "format": "slack",
  "options": {},
  "includeAttachments": true,
  "mailboxId": "r_abcd1234",
  "email": "r_abcd1234@hook.echovalue.dev",
  "hash": "a1b2c3d4e5f6..."
}
```

| Field | Type | Description |
| ----- | ---- | ----------- |
| `name` | string | Webhook name |
| `webhook` | string | Configured callback URL |
| `headers` | object | Custom headers |
| `format` | string | Native format, if configured (e.g. `slack`, `telegram`, `custom`) |
| `template` | object | Custom template object (only present when format is `custom`) |
| `options` | object | Format options (sensitive values are redacted) |
| `includeAttachments` | boolean | Whether attachments are included in webhook payload |
| `mailboxId` | string | Opaque mailbox identifier for management operations |
| `email` | string | Opaque mailbox address (`<mailboxId>@hook.echovalue.dev`) |
| `hash` | string | SHA256 hash of your token |

**Response headers:**

| Header | Description |
| ------ | ----------- |
| `x-balance` | Wallet balance after this call |
| `x-cost` | Tokens consumed (1) |

See [Response Headers](/response-headers) for details.

| Status | Meaning |
| ------ | ------- |
| `200` | Webhook saved |
| `400` | Invalid URL or malformed request |
| `401` | Invalid token |
| `402` | Insufficient credits |

:::caution
The webhook URL must be HTTPS and publicly reachable. Localhost and private IP addresses are not accepted.
:::
