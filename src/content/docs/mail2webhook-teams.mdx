When `format` is set to `teams`, incoming emails are converted into Microsoft Teams messages using the [Incoming Webhook connector](https://learn.microsoft.com/en-us/microsoftteams/platform/webhooks-and-connectors/how-to/add-incoming-webhook) format. No custom payload handling required on your end.

---

## Get the Webhook URL

Microsoft Teams supports incoming webhooks via **Workflows** (the newer method) or **Connectors** (legacy). Use Workflows if available in your tenant.

### Via Workflows (recommended)

<Steps>

1. In the channel, click **+** next to the message input → **Workflows**.

2. Search for **"Post to a channel when a webhook request is received"** and select it.

3. Follow the setup wizard. At the end, copy the generated webhook URL — it looks like:
   ```
   https://prod-00.westeurope.logic.azure.com:443/workflows/XXXX/triggers/manual/...
   ```

</Steps>

### Via Connectors (legacy)

<Steps>

1. In the channel, click **···** → **Connectors**.

2. Search for **Incoming Webhook**, click **Add** → **Configure**.

3. Give it a name and click **Create**. Copy the generated URL.

</Steps>

## Configure the Webhook

```sh
curl 'https://api.echovalue.dev/webhook' \
  -H 'x-token: mytoken' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://prod-00.westeurope.logic.azure.com:443/workflows/XXXX/...",
    "format": "teams"
  }'
```

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `url` | string | Yes | Your Teams Webhook URL |
| `format` | string | Yes | Must be `"teams"` |
| `options` | object | No | Not used for Teams |

---

## Message Preview

Each email arrives in Teams as:

```
**Subject line**
From: Sender Name <sender@example.com>
Email body text
```

:::note
The plain text body is used. HTML is stripped automatically.
:::
