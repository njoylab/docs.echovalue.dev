The API uses standard HTTP status codes. Error responses are returned as plain text with a short description.

**Example error response:**

```
Key Not Found
```

:::note
Requests that result in errors are still counted as API calls and consume credits on paid endpoints, to prevent misuse. `GET /token/logs` remains free.
:::

## 4xx Client Errors

| Code | Name | When it occurs |
| ---- | ---- | -------------- |
| `400` | Bad Request | The request is malformed or missing required parameters |
| `401` | Unauthorized | The `x-token` header is missing or invalid |
| `402` | Payment Required | Your wallet has no credits remaining |
| `404` | Not Found | The key, webhook, or resource does not exist |
| `405` | Method Not Allowed | The HTTP method is not supported for this endpoint |

## 5xx Server Errors Documented In OpenAPI

| Code | Name | When it occurs |
| ---- | ---- | -------------- |
| `500` | Internal Server Error | Unexpected server-side failure — try again later |

## Common Scenarios

| Situation | Status code |
| --------- | ----------- |
| Missing `x-token` header | `401` |
| Token doesn't exist | `401` |
| No credits left | `402` |
| GET on a key that was never set | `404` |
| GET webhook list when none is configured | `200` with an empty `webhooks` array |
| DELETE webhook with missing or unknown `mailboxId` | `400` or `404` |
| Calling an endpoint with the wrong HTTP method | `405` |

## Notes

- `GET /token/logs` is free. The current OpenAPI file does not declare `x-cost` or `x-balance` headers for this endpoint.
- Infrastructure in front of the application may still emit `429 Too Many Requests` or `503 Service Unavailable`, but those responses are not currently modeled in `openapi.yaml`.
