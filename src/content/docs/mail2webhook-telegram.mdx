When `format` is set to `telegram`, incoming emails are sent to a Telegram chat via the [Bot API](https://core.telegram.org/bots/api#sendmessage) using HTML formatting. This format requires two things: a bot token (embedded in the URL) and a `chat_id` option.

---

## Get the Bot Token and Chat ID

### Step 1 — Create a bot

<Steps>

1. Open Telegram and search for **@BotFather**.

2. Send `/newbot` and follow the instructions to choose a name and username.

3. BotFather will reply with your **bot token**:
   ```
   1234567890:ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghi
   ```

</Steps>

### Step 2 — Get your Chat ID

<Steps>

1. Send any message to your bot (you must message it first, or it can't reply to you).

2. Open this URL in your browser (replace `{TOKEN}` with your bot token):
   ```
   https://api.telegram.org/bot{TOKEN}/getUpdates
   ```

3. Find the `chat.id` value in the JSON response:
   ```json
   {
     "message": {
       "chat": {
         "id": 123456789
       }
     }
   }
   ```

</Steps>

:::tip
To forward emails to a **group or channel**, add the bot to it and send a message there. The `chat.id` for groups is a negative number (e.g., `-1001234567890`).
:::

## Configure the Webhook

The `url` is always `https://api.telegram.org/bot{TOKEN}/sendMessage` with your actual token substituted in.

```sh
curl 'https://api.echovalue.dev/webhook' \
  -H 'x-token: mytoken' \
  -H 'Content-Type: application/json' \
  -d '{
    "url": "https://api.telegram.org/bot1234567890:ABCDEFGHIJKLMN/sendMessage",
    "format": "telegram",
    "options": {
      "chat_id": "123456789"
    }
  }'
```

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `url` | string | Yes | `https://api.telegram.org/bot{TOKEN}/sendMessage` |
| `format` | string | Yes | Must be `"telegram"` |
| `options.chat_id` | string | Yes | The Telegram chat, group, or channel ID |

## Message Preview

Each email arrives in Telegram as an HTML-formatted message:

```
<b>Subject line</b>
From: Sender Name <sender@example.com>
Email body text
```

:::note
Special HTML characters in the subject, sender name, and body are escaped automatically to prevent rendering issues.
:::
