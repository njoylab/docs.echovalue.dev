`GET https://api.echovalue.dev/token`

Returns remaining credits and token metadata.

:::note
**Cost:** 1 credit. For ongoing monitoring, prefer the `x-balance` response header on paid endpoints.
:::

## Request

**Headers:**

| Header | Description |
| ------ | ----------- |
| `x-token` | Your API token |

## Response

`200 OK` — JSON object.

```json
{
  "wallet": 12345,
  "created": "2023-08-09T15:40:09.77Z",
  "hash": "a1b2c3d4e5f6..."
}
```

| Field | Type | Description |
| ----- | ---- | ----------- |
| `wallet` | integer | Remaining credit balance |
| `created` | string (ISO 8601) | Date when the token was created |
| `hash` | string | SHA256 hash of your token |

**Response headers:**

| Header | Description |
| ------ | ----------- |
| `x-balance` | Wallet balance after this call |
| `x-cost` | Credits consumed |

See [Response Headers](/response-headers/) for details.

## Status Codes

| Status | Meaning |
| ------ | ------- |
| `200` | Balance returned |
| `401` | Invalid token |
| `402` | Insufficient credits |

## Notes

- `hash` is the SHA256 hash of the wallet token and can be used for identification in downstream payloads.

## Examples

```sh
curl 'https://api.echovalue.dev/token' \
  -H 'x-token: mytoken'
```
